ACCEPT Real Time Plug-in

Use Case:

In this example, there are three paragraphs that will be used to demonstrate:
  • - Multi placeholder functionality, by instantiating the plug-in pointing to multiple paragraphs (HTML "P" elements).
  • - The Real Time environment(using Fiddler or other tool for monitoring is advisable).
  • - The versatility of the plug-in when displaying flags inline with the multiple placeholders(again, in this case HTML "P" elements).
The following two first paragraphs will be scoped by the plug-in configuration, so when clicking the "Trigger..." button, it triggers a check against the configured ACCEPT API endpoint and displays two flags: - one spelling error flagged(mouse over "testt" it to display the context-menu) in the first paragraph. - one grammar rule is flagged(mouse over "an grammar" it to display the context-menu) in the second paragraphs.

Placeholders:

this one contains a testt spelling error

this one contains an grammar error

this one doesn't contain the CSS attribute class="process-me". The plug-in will not scope it...
